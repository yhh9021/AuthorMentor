## 最新修正

- 支持 `1.第1章` 这类电子书章节标题格式，修正《诡秘之主》章节解析。
- 素材库来源字段已改为当前书名，不再错误写死为《我有一个修仙世界》。
- 修正了跨书题材污染：非仙侠作品不再使用“道院、水府、制符、筑基、元婴、魔主”等仙侠专属分段标签。
- 为每本书增加类型画像、设定沉淀、优点机制、高光片段和质量审计。
- 拆书能力已增加深度设定、深度优点机制、深度高光片段三类产物。
- 高光数量改为按本地章节体量动态确定，并结合正文、章节索引和外部讨论信号选取候选片段。
- 优点机制不足 5 条时，会从本书核心优点和高光维度补足，避免输出过少。
- 每本书新增独立设定集，覆盖修炼/能力体系、地图、势力、资源、人物身份、世界规则、设定时间线和复用边界。
- 每本书持久拆书结果改为单层中文文件；旧的 `book-map/`、`segments/`、`material-cards/`、`synthesis/` 子目录已不再作为提交产物。
- 素材库更新改为按来源替换，重跑同一书同一章节范围时不会无限追加重复条目。
- 原文副本继续保存在各书 `source/` 目录和 `runs/*/input/` 目录，这些目录已被 Git 忽略，没有提交到远程。
- `deep-run` 已改造成子 Agent 精读架构：先生成 `runs/<run-id>/input/agent-chunks/NNNN.md`，再合并 `runs/<run-id>/output/agent-chunks/NNNN.json`；未提供外部 Agent 输出时，会保留脚本兜底抽取，但审计会标记“子 Agent 精读覆盖率”需返工。
- 深拆产物新增 `关键事件链.md`，用于承接子 Agent JSON 里的关键事件、证据摘要和后续影响。
- `人物与关系图.md` 已改为语义关系版：先输出主角候选、人物功能分层、主角关系网、主要矛盾、同盟合作、师徒/上下级/亲属/交易关系，再把低置信共现放入“待精读确认”。
- 人物抽取增加降噪和综合主角评分，避免“转而、开口、段时间、明显”等动作/副词/普通词进入主角判断。
