# 生产能力采用 Skill 优先结构

拆书等生产能力先沉淀为可被智能体阅读和执行的 Skill，而不是写死在某个脚本或某个智能体 CLI 中。TypeScript CLI 负责生成任务包、校验输出、应用结果和提交 Git；Codex、Claude Code 或其他智能体只是可替换的执行者。

**考虑过的方案**

- 直接在脚本中写死提示词并调用某个智能体
- 每个能力先沉淀为 Skill，再由 CLI 编排运行

**结果**

能力目录包含 `SKILL.md`、`references/`、`templates/` 和可选脚本。每次能力调用生成一个 `runs/<run-id>/` 任务包，任务包包含任务说明、输入、上下文、输出契约和产物目录。
